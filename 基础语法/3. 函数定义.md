# 🐍 Python vs 🌐 JavaScript 函数定义对比

## 1️⃣ 普通函数定义

### Python

```python
def add(x, y):
    return x + y

print(add(2, 3))  # 5
```

### JavaScript

```javascript
function add(x, y) {
  return x + y;
}

console.log(add(2, 3)); // 5
```

---

## 2️⃣ 匿名函数（lambda / 箭头函数）

### Python

```python
square = lambda x: x * x
print(square(4))  # 16
# lambda 参数1, 参数2, ...: 表达式(返回值,单行)
```

👉 `lambda` 只能写单表达式，不能写多行语句。

### JavaScript

```javascript
const square = (x) => x * x;
console.log(square(4)); // 16
```

👉 箭头函数可多行，但要写 `{}` 和 `return`。

---

## 3️⃣ 高阶函数应用

### Python

```python
nums = [1, 2, 3]
print(list(map(lambda x: x * 2, nums)))  # [2, 4, 6]
```

### JavaScript

```javascript
let nums = [1, 2, 3];
console.log(nums.map(x => x * 2)); // [2, 4, 6]
```

---

## 📌 总结对比表

| 功能       | Python                      | JavaScript                   |
| -------- | --------------------------- | ---------------------------- |
| **函数定义** | `def func(...):`            | `function func(...) {}`      |
| **匿名函数** | `lambda args: expr`         | `(args) => expr`             |
| **多行函数** | 只能用 `def`                   | 箭头函数可多行 `{ ... }`            |
| **高阶函数** | 常配合 `map`/`filter`/`reduce` | 同样配合 `map`/`filter`/`reduce` |
| **可读性**  | `def` 清晰，`lambda` 简洁        | `function` 传统，`=>` 简洁        |
